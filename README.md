Vyu opens images, videos, audios, and more — with real editing, clipping, converting, and format support built in. No telemetry, no accounts, no extensions needed. Everything works out of the box and stays on your machine offline. It's the much needed replacement for Windows Photos and ancient-looking viewers.

## Features

### Images

- **Format support** — JPG, PNG, GIF, WebP, BMP, SVG, AVIF, HEIC/HEIF, TIFF, PSD, JXL, and 24 RAW formats (CR2, CR3, NEF, ARW, DNG, RAF, RW2, and more).
- **Auto-decode** — TIFF, PSD, JXL, HEIC, and RAW files are decoded by the backend and cached as PNGs. Instant on second open.
- **Zoom, pan, rotate, flip** — scroll to zoom, drag to pan, keyboard shortcuts for rotation and mirroring.
- **Adjustments** — brightness, contrast, saturation, and hue with live preview.
- **Crop** — drag handles with aspect-ratio lock.
- **Clipboard** — copy an image out or paste one in to open it.

### Video

- **Format support** — MP4, WebM, MKV, AVI, MOV, WMV, MPEG, M4V, TS, M2TS (TS/M2TS remuxed on the fly).
- **Frame-accurate timeline** — scrubber with markers, clip boundaries, and A-B loop.
- **Timestamp bookmarks** — drop labelled markers on a video, saved per file.
- **Clip export** — mark in/out points, export separately or merged.
- **Playback speed** — 0.25x to 3x.
- **Frame stepping** — comma and period nudge one frame at a time.
- **Copy current frame** — grab a PNG from the context menu.

### Audio

- **Format support** — MP3, WAV, FLAC, OGG, AAC, WMA, M4A, Opus, AIFF, ALAC.
- **Waveform thumbnails** — generated by FFmpeg so you can see the shape of a track before opening it.
- **Loop modes** — loop, stop, auto-advance, or shuffle.
- **Volume** — stepped buttons or continuous slider, with mute.

### PDFs

- **Page-by-page rendering** — code-split, only loads when you open a PDF.
- **Zoom and pan** — same controls as the image viewer.

### Across the app

- **Slideshow** — configurable interval, next or shuffle, fade/slide/cut transitions. PDFs skipped automatically.
- **Themes** — dark, light, or system. Animated transitions.
- **Custom title bar** — filename inline, rename in place.
- **Drag-and-drop** — drop files to open them.
- **Right-click menus** — copy, copy path, show in Explorer, properties, delete, trash.
- **Thumbnail bar** — browse siblings in the same folder.
- **Sort** — name, date modified, date created, size, or type.
- **Window state** — position, size, and maximized state persist across restarts.
- **Convert & compress** — convert between formats with presets, or zip a folder.
- **Media integrity check** — detect and attempt repair via FFmpeg.
- **File properties** — codec, dimensions, bit depth, frame rate, and more.
- **Accessibility** — larger text, reduced motion, and other options in the accessibility dialog.
- **Keyboard-driven** — arrows to browse, Space to play, F for fullscreen, and full keybind coverage.
- **FFmpeg auto-install** — if FFmpeg isn't on your PATH, Vyu offers to install it via winget.

# For developers

## Design decisions

A few choices shaped Vyu's architecture. Worth knowing before you start changing things.

**Tauri, not Electron.** Small binary, low memory footprint, direct file system access without bundling a second browser engine. The trade-off is a Rust toolchain as a prerequisite.

**SvelteKit as a static SPA.** `adapter-static` with `fallback: "index.html"` — no SSR, no server, just a single-page app in the Tauri webview. Simple pipeline, fully testable in a browser.

**Svelte 5 runes, no legacy syntax.** `$state`, `$derived`, `$effect` throughout. No `let` / `$:`.

**localStorage for persistence.** All user state — volume, theme, timestamps, clip boundaries, loop mode, resume points — lives in localStorage with a `vyu-` prefix. No backend, no database. State is tied to the browser profile, so wiping browser data wipes Vyu's state too. For a single-user desktop app, that's a fair trade.

**FFmpeg on PATH, not bundled.** Vyu shells out to `ffmpeg` and `ffprobe` for thumbnails, remuxing, waveforms, integrity checks, conversion, and clip export. Bundling FFmpeg would bloat the installer; PATH keeps it lean. Vyu checks on startup and offers a winget install if it's missing.

**No router.** Intentionally no SvelteKit routes. All UI lives in `src/routes/+page.svelte` and `src/lib/`. Routes would just add complexity with no real benefit at this scale.

**Custom title bar.** `decorations: false` in `tauri.conf.json` — Vyu draws its own so it blends with the theme and puts the filename there for inline rename.

**Features live in their own modules.** Each concern — viewer, editing, playback, clips, PDF, markup, theme, file watching — is a self-contained module under `src/lib/features/`. `+page.svelte` is a layout shell that wires them together; new state and handlers belong in a feature module, not in the page.

## Tech stack

| Layer                    | Tool                                    |
| ------------------------ | --------------------------------------- |
| Desktop shell            | Tauri 2                                 |
| UI framework             | Svelte 5 (runes mode)                   |
| Meta-framework           | SvelteKit (static SPA, no SSR)          |
| Language                 | TypeScript (strict)                     |
| Package manager          | pnpm                                    |
| Video / image processing | FFmpeg (system PATH, not bundled)       |
| PDF rendering            | pdfjs-dist (code-split, dynamic import) |
| State persistence        | localStorage                            |

## Prerequisites

- **Rust toolchain** — [rustup.rs](https://rustup.rs)
- **Node.js + pnpm** — `npm install -g pnpm` or [corepack](https://nodejs.org/api/corepack.html)
- **FFmpeg** — not bundled; Vyu will offer to install via winget on first launch if missing

## Commands

| Task       | Command                   |
| ---------- | ------------------------- |
| Install    | `pnpm install`            |
| Dev        | `pnpm tauri dev`          |
| Build      | `pnpm tauri build`        |
| Type check | `pnpm check`              |
| Format     | `pnpm prettier --write .` |

Dev server is pinned to port **1420** (`strictPort: true`). Make sure it's free.

## Project structure

```
src/                    # Frontend (Svelte + TypeScript)
├── routes/
│   ├── +page.svelte    # Layout shell — wires feature modules together
│   └── +layout.ts      # Disables SSR
├── lib/
│   ├── features/       # Feature modules: media, viewer, timeline, editing, pdf, navigation, dialogs, menus
│   ├── services/       # Folder scanning, localStorage, clipboard, context menus
│   ├── shared/         # Constants, types, keybinds, tooltip, shell, toast
│   └── styles/         # Modular CSS — variables, layout, components, overlays, animations
└── static/             # Static assets

src-tauri/              # Backend (Rust)
├── src/
│   ├── lib.rs          # run() + setup() — ~170 lines
│   ├── constants.rs    # Extension lists, magic numbers
│   ├── types.rs        # Shared structs (MediaKind, ThumbState, etc.)
│   ├── util.rs         # Helpers (run_ffmpeg, hash_path_xxh3, check_cache, etc.)
│   └── commands/       # One file per domain: thumbnail, display, editing, conversion, clips, file_ops, clipboard, integrity, external_apps
├── tauri.conf.json     # App config, file associations, bundle
├── Cargo.toml          # Rust dependencies
└── capabilities/       # Tauri 2 capability permissions
```

Module ownership map lives in [ARCHITECTURE.md](./ARCHITECTURE.md).
